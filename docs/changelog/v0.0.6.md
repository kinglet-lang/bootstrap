## v0.0.6

### Language Features

- **Cast expression** ‚Äî `int(x)`, `float(x)`, `string(x)` replace the previous `s.to_int()` / `s.to_float()` methods. Infallible casts (int‚Üîfloat, numeric‚Üístring) produce the target type directly; fallible casts (string‚Üíint, string‚Üífloat) produce a `CastError` variant on failure.
- **`?:` coalesce operator** ‚Äî `e ?: f` evaluates `f` only when `e` is null or a `CastError`. Right-associative; chains naturally: `a ?: b ?: c`.
- **`?:` let-binding form** ‚Äî `e ?: let err => f` binds the `CastError` payload to `err` for inspection in the fallback expression. Requires the LHS to be a fallible cast.
- **Postfix `?` propagation** ‚Äî `e?` inside a `T?`-returning function returns null early on failure; inside a `try` block transfers control to the matching `catch`.
- **`try` / `catch`** ‚Äî `try { ... } catch (let e: CastError) { ... }` catches propagation from `?` inside the try body. The error value is bound to a local variable in the catch body.
- **Built-in `CastError` enum** ‚Äî `CastError { Empty, NotANumber(string), Overflow(string) }` is pre-registered in every program. `int(s)` and `float(s)` produce the appropriate variant on failure.

### Removed

- `s.to_int()` / `s.to_float()` string methods ‚Äî replaced by `int(s)` / `float(s)` cast expressions
- `StringToInt` / `StringToFloat` opcodes ‚Äî replaced by the unified `CastTo` opcode
